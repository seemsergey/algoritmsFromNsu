
# Minimum Cost Arborescence (Алгоритм Эдмондса)

Данный проект представляет собой реализацию алгоритма для нахождения **минимального ориентированного ветвления** (Minimum Cost Arborescence) в ориентированном графе.

## Задача
В компании работает $N$ сотрудников. Один из них — директор ($R$). Для каждой пары "наставник -> подчинённый" задана стоимость оформления связи. Необходимо построить такую структуру управления, чтобы:
1. У каждого сотрудника (кроме директора) был **ровно один** наставник.
2. От директора можно было добраться до любого сотрудника.
3. Суммарная стоимость всех связей была **минимальной**.

Если построить такую структуру невозможно, программа выводит `IMPOSSIBLE`.

## Алгоритм
Решение разбито на несколько этапов:

1. **Проверка связности (BFS):**
    - Используется поиск в ширину для проверки, достижимы ли все $N$ вершин из корня $R$.
    - Для оптимизации поиска используется формат **CSR (Compressed Sparse Row)**, что позволяет выполнять проверку за $O(V + E)$.

2. **Алгоритм Эдмондса (Чу-Лю):**
    - **Жадный выбор:** Для каждой вершины (кроме корня) выбирается входящее ребро с минимальным весом.
    - **Поиск циклов:** Проверяется наличие циклов в выбранном наборе ребер.
    - **Сжатие (Contraction):** Если найдены циклы, они "стягиваются" в супер-вершины, веса входящих в них ребер пересчитываются, и алгоритм запускается повторно для сжатого графа.
    - **Базовый случай:** Когда циклов не остается, накопленная сумма весов является ответом.

## Технические особенности
- **Типы данных:** Для хранения весов и итоговой суммы используется `long long`, так как веса ребер могут достигать $10^9$, а их сумма может превысить возможности 32-битного `int`.
- **Эффективность:** Алгоритм подходит для графов с $N \le 200$ и $M \le 2000$, работая практически мгновенно.
- **Память:** Вся динамическая память (массивы для CSR, очереди BFS, метки циклов) корректно освобождается.

## Формат входных данных
Первая строка содержит три целых числа:
- `N` — количество сотрудников.
- `M` — количество возможных связей.
- `R` — номер директора.

Следующие `M` строк содержат описания связей:
- `u v w` — связь от `u` к `v` стоимостью `w`.

## Пример использования

### Ввод
```text
3 3 0
0 1 10
1 2 10
0 2 30
```

### Вывод
```text
20
```
